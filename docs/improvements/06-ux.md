# UX improvements

Ordered roughly by (user value ÷ effort). Items already tracked in
[ROADMAP.md](../../ROADMAP.md) are marked with their backlog IDs.

## 1. Extra-keys row in the terminal (P0)

Esc, Tab, Ctrl (sticky), Alt, arrows, `|`, `/`, `-`, plus long-press variants.
Detailed in [doc 2 §2](02-terminal-emulation.md) — listed here because it is
the most user-visible quick win in the whole review: today `↑` history recall,
`Ctrl-R`, and exiting `vi` are all impossible from the on-screen keyboard.

## 2. Terminal readability controls (P1)

- **Font size**: pinch-to-zoom plus a persisted setting (re-report PTY size on
  change — the `onSizeChangedListener` plumbing already exists).
- **Color scheme**: at minimum honour dark/light theme in terminal colors; the
  ANSI palette in `TerminalView.parseAnsi` uses raw `Color.RED`-style constants
  that are hard to read on both themes (pure blue on black is the classic
  offender). Define the 16-color palette in resources per theme.
- **Keep-screen-on toggle** while a session is active — standard for terminal
  apps, one `FLAG_KEEP_SCREEN_ON` away.

## 3. Scrollback: search, share, jump (P1)

- Search within the session buffer (`getRawText()` already exposes it) with
  match highlighting and next/prev.
- "Share/export session log" action — plumbing exists via
  `TerminalLogRepository` / Drive uploads; a simple `ACTION_SEND` of the raw
  buffer covers the local case.
- "Jump to bottom" FAB when the user has scrolled up; today new output forces
  scroll-to-bottom (`updateText`'s `post { scrollTo(...) }`) even while the
  user is reading history — that forced scroll should be suppressed whenever
  the view is not already at the bottom.

## 4. Session presence & multi-session (P1/P2)

- Connection state chip (connected/reconnecting/lost + duration) visible in
  `TerminalActivity` — pairs with the reconnect work in
  [doc 3](03-connection-reliability.md).
- Multiple simultaneous sessions with a switcher ([doc 3 §5](03-connection-reliability.md)).
- Play finish notification (**B-4**) — long-running Plays currently require
  staring at the screen.

## 5. Host & phrase management polish (P2)

- Phrase search/filter (**B-1**) and quick-access phrase slots in the terminal
  (**B-3**) — both already in the backlog, both cheap.
- Host list: show reachability at a glance (last-connected timestamp; optional
  async TCP probe), pin a default host (**B-8**).
- Host export/import (encrypted JSON, passwords optional) — today migrating to
  a new phone means retyping every host, since backups intentionally exclude
  secure prefs.

## 6. Input affordances (P2)

- **Password field paste bug class**: verify paste works in all credential
  fields (common complaint with `textPassword` inputs).
- Physical keyboard support beyond letters ([doc 2 §2](02-terminal-emulation.md)).
- Optional haptic feedback on extra-keys presses.

## 7. Accessibility & platform fit (P2/P3)

- TalkBack: the terminal view is a giant mutating `TextView`; mark output
  updates with `accessibilityLiveRegion="polite"` and label the Ctrl-C/D
  buttons (`contentDescription`).
- Material 3 dynamic color (Monet) for the app chrome — the terminal itself
  keeps its own palette.
- Predictive back + edge-to-edge insets (target SDK 36 already; verify IME
  insets in the terminal don't overlap the extra-keys row once added).
- Localization is in good shape (fi/sv/es) — add a CI check that new string
  keys exist in all locales, or mark them `translatable="false"` explicitly.

## 8. First-run and empty states (P3)

The setup checklist card is a strong start. Remaining rough edges:

- Empty terminal tab before any host exists could offer a one-tap "try the
  local shell" (the synthetic local host already exists —
  `seedLocalHostIfMissing`).
- After a failed connect, the error banner offers retry; add "edit this host"
  as a second action for auth failures, deep-linking into `HostEditActivity`.
